Prompt：進階 2D 幾何 Tutting 舞蹈模擬器（七頭身站姿專業版）
🎯 專案目標
請建立一個單一網頁檔案（Single HTML File）的「極簡主義風格 Tutting 舞蹈火柴人模擬器」。此工具專為 Tutting 街舞愛好者與空間幾何視覺設計師打造，用於編舞、空間方框疊加（Box Folding）與動作練習。專案必須完全使用原生 HTML5 Canvas、CSS3 與 Vanilla JavaScript 實作，嚴禁引入任何第三方程式庫（如 Three.js, p5.js, GSAP 或外部 IK 庫）。
🛠 核心技術規格與架構
1. 視覺與行動端 UI/UX 設計
視覺風格： 極簡主義（Minimalism）、高對比高冷時尚感。背景使用極深色（#08080a），主骨架線條使用純白（#ffffff），末端控制點與狀態提示使用螢光青綠色（#00ffcc）。
幾何基準線： 在畫布中央水平與垂直交叉處繪製極暗的十字基準線（#111116），輔助使用者對齊直角。
行動端手勢優化： * 採用 RWD 上下佈局：上方為畫布，下方為控制面板。
所有按鈕與切換開關的點擊區域必須符合行動端觸控規範（至少 44x44 px，建議高度 48px），並具備優雅的 :active 觸控變色反饋。
針對手機觸控，必須將核心關節點的點擊判定半徑（Hit Test Radius）擴大至 26px，以確保手指拖曳時靈敏、流暢且防誤觸。
2. 真實系「七頭身」黃金幾何比例
為了消除卡通感並確保完美的「箱型摺疊（Box Folding）」視覺效果，人體骨骼長度（單位：像素）必須嚴格依據數學計算設定如下（頭高為直徑 30px，總身高剛好為 7 倍頭高，即 210px）：
HEAD_RADIUS（頭部半徑）: 15 （即頭高為 30px，採空心圓設計）
NECK_LEN（頸部長度）: 10
TORSO（軀幹/脊椎長度）: 60
SHOULDER（單側肩寬）: 26 （窄肩設計，確保雙臂動作的幾何中心更凝聚）
HIP_WIDTH（單側髖寬）: 18
UPPER_ARM（大臂）: 50
FOREARM（前臂）: 50 (※ 大前臂 1:1 對稱，確保 180° 折疊時線條完美重合)
HAND（手掌長度）: 20 (※ 自然下垂時指尖剛好落在大腿中段)
THIGH（大腿長度）: 55
CALF（小腿長度）: 55 (※ 大小腿 1:1 對稱，確保極致深蹲時方框完美對齊)
FOOT（腳掌長度）: 20
3. 開局初始姿勢（預設動作設定）
火柴人啟動時的預設姿態必須是標準的直立站姿結合對稱手部幾何：
下半身（直立站姿）： 雙腿髖關節垂直向下（角度為 90°），膝關節打直不彎曲（相對角度為 0），雙腳腳掌自然水平貼地。
上半身（幾何手勢）： 雙臂呈現經典的對稱 Tutting 折線。左臂（肩膀 180°、手肘 -90°、手腕 90°）；右臂（肩膀 0°、手肘 90°、手腕 -90°）。
畫布錨點： 配合七頭身高度，將角色物理中心（Pelvis）垂直向上修正，確保角色在全螢幕或不同解析度下皆能完美居中、不踩到底部面板。
4. 全身雙模式互動系統：FK（正向運動學）與 IK（逆向運動學）
介面需提供明顯的「FK 模式」與「IK 模式」切換開關（Toggle）：
FK 模式（Forward Kinematics）：
使用者可獨立點擊並拖曳全身 12 個核心關節。
手臂鏈：肩膀 $\rightarrow$ 手肘 $\rightarrow$ 手腕 $\rightarrow$ 手掌。
腿部鏈：髖部 $\rightarrow$ 膝蓋 $\rightarrow$ 腳踝 $\rightarrow$ 腳掌。
拖曳單一關節時，僅改變該關節相對於父階層的相對角度，其子關節隨之產生對應的矩陣旋轉與連動。
IK 模式（Inverse Kinematics）：
手臂鏈結： 以「肩膀」為固定基底（Base），「手腕/手掌」為末端效應器（End Effector）。
腿部鏈結： 以「髖部」為固定基底（Base），「腳踝/腳掌」為末端效應器（End Effector）。
演算法要求： 使用 FABRIK (Forward And Backward Reaching Inverse Kinematics) 演算法實作 2D 空間中的兩節骨骼反向解算。當使用者拖曳手腕或腳踝時，自動計算並更新大臂/前臂（或大腿/小腿）的相對角度，實現末端追隨。
邊界安全锁： 當觸控點超出四肢的最大伸展範圍時，系統必須將骨骼鎖定在朝向目標點的最大直線路徑上，防止骨骼拉伸分離或脫臼。
5. 完美 Tutting 角度吸附機制 (Angle Snapping)
無論在 FK 還是 IK 模式下，當任何關節的絕對或相對角度接近 0°, 45°, 90°, 135°, 180°, 225°, 270°, 315° 的倍數時（容許誤差範圍 $\pm5^\circ$），系統必須自動將角度吸附（Snap）至該精準角度。
視覺與狀態提示： 觸發吸附時，當前互動側的關節點需從白色轉為螢光青綠色（#00ffcc）並擴大半徑閃爍，頂部狀態列的 SNAP 顯示需同步切換為亮綠色的 TRIGGERED。
6. 影格紀錄與 LERP 動畫時間軸引擎
資料結構： 時間軸必須統一儲存各關節的最終相對角度（Relative Angles），確保不論用 FK 還是 IK 擺出的姿勢，都能以完全相同的資料格式被正確記錄。
時間軸 UI 功能： 提供「▶ 播放 / ⏸ 暫停」、「＋ 新增影格」、「🗑 刪除目前影格」、「✕ 清除全部」按鈕，並動態生成可點擊跳轉的影格節點。
Time-based LERP 動畫引擎：
動畫播放必須基於時間差（Time-based Delta Time）平滑計算影格之間的過渡，避免受瀏覽器刷新率（FPS）波動影響。
臨界點優化（最短路徑角度內插）： 必須撰寫優化過的角度線性內插演算法（利用 Math.cos 與 Math.sin 處理差值），嚴格禁止四肢在跨越 $+180^\circ$ 與 $-180^\circ$ 邊界臨界點時發生原地瘋狂自轉圈圈的 Bug。
動畫需循環播放（Loop）。在非播放且未拖曳狀態下，手動切換不同關鍵影格時，火柴人也應平滑內插過渡至該影格姿態。
📦 交付格式與程式碼質量
請將所有的 HTML 標記、CSS 樣式表與 JavaScript 邏輯程式碼整合在一個完整的、無任何外部依賴的 .html 檔案中。JavaScript 代碼中必須針對 FABRIK 反向解算、相對角度回推、角度幾何吸附（Snap）、以及跨越 $\pm\pi$ 邊界的角度 LERP 處理 補上詳盡的開發註解。
